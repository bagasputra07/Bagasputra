PENGERTIAN OOP

Object Oriented Programming (OOP) tidak hanya sekadar teknik pemrograman, tetapi merupakan pendekatan desain sistem yang meniru cara manusia memahami dunia nyata. Dalam kehidupan sehari-hari, kita mengenali entitas sebagai objek yang memiliki karakteristik dan perilaku. OOP mengadopsi konsep ini ke dalam pengembangan perangkat lunak sehingga struktur program menjadi lebih intuitif dan mudah dipahami.

OOP bertujuan untuk meningkatkan modularitas sistem, yaitu dengan memecah program besar menjadi bagian-bagian kecil yang terorganisir dalam bentuk class. Setiap class bertanggung jawab atas satu konsep atau entitas tertentu, sehingga pengembangan dan pemeliharaan sistem menjadi lebih terstruktur.

Selain itu, OOP mendukung prinsip reusability atau penggunaan ulang kode. Class yang telah dibuat dapat digunakan kembali dalam proyek lain tanpa perlu menulis ulang logika yang sama. Hal ini sangat penting dalam pengembangan perangkat lunak modern yang menuntut efisiensi waktu dan konsistensi kualitas.

Dalam praktiknya, OOP memiliki empat pilar utama, yaitu Encapsulation, Abstraction, Inheritance, dan Polymorphism. Keempat prinsip ini memungkinkan sistem menjadi lebih fleksibel, aman, dan mudah dikembangkan. Meskipun pada laporan ini lebih difokuskan pada Encapsulation dan Constructor, Dart sebagai bahasa pemrograman mendukung seluruh pilar tersebut secara native.

Dengan menerapkan paradigma OOP, pengembang dapat membangun sistem yang scalable (mudah dikembangkan), maintainable (mudah dirawat), dan robust (tahan terhadap kesalahan). Pendekatan ini menjadi fondasi utama dalam pengembangan aplikasi modern, termasuk framework Flutter yang menggunakan Dart sebagai bahasa utamanya.

Pada source code ini, class Car memiliki dua atribut yaitu model dan color. Method start() dan stop() menggambarkan perilaku objek mobil. Class ini menunjukkan konsep dasar OOP yaitu pembuatan objek dari sebuah blueprint (class).

1. CLASS DAN OBJECT


class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }

  void stop() {
    print('Car stopped!');
  }
}

Constructor digunakan untuk memberikan nilai awal saat objek dibuat. Dengan Car(this.model, this.color), setiap objek Car akan langsung memiliki nilai model dan color ketika diinisialisasi. Ini meningkatkan efisiensi dan memastikan data tidak kosong.

2. CONSTRUCTOR


class Car {
  String? model;
  String? color;

  Car(this.model, this.color);

  void start() {
    print('Car started!');
  }
}

3. VALIDASI DATA

Class Mahasiswa menerapkan validasi pada constructor. Jika nilai IPK berada di luar rentang 0.0–4.0, program akan menghasilkan error. Ini merupakan contoh penerapan kontrol data agar sistem lebih aman dan robust. 


class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk) {
    if (ipk < 0.0 || ipk > 4.0) {
      throw ArgumentError('IPK harus 0.0 - 4.0');
    }
  }
}

Atribut name diawali dengan underscore () yang menandakan bahwa variabel tersebut bersifat private. Akses dilakukan melalui method setName() dan getName(). Konsep ini disebut Encapsulation, yaitu membatasi akses langsung ke data.

4. ENCAPSULATION


class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  void setName(String name) {
    _name = name;
  }

  String getName() {
    return _name;
  }
}

Constructor dengan named parameter membuat kode lebih jelas dan fleksibel. Kata kunci required memastikan nilai wajib diisi, sementara nilai default seperti stok = 0 dan kategori = 'Umum' memberikan keamanan tambahan jika tidak diinput.

5. NAMED PARAMETER


class Produk {
  final String id;
  final String nama;
  final double harga;
  final int stok;
  final String kategori;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
    this.stok = 0,
    this.kategori = 'Umum',
  });
}
